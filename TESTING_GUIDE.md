# Руководство по тестированию

Это руководство описывает, как тестировать различные аспекты развертывания приложения, включая CI/CD пайплайн и локальную установку.

## 1. Тестирование CI/CD пайплайна

Пайплайн включает этапы сборки, валидации и развертывания.

### А. Тестирование этапа валидации (`validate`)

Этот этап проверяет синтаксис и корректность ваших Kubernetes-манифестов с помощью Kustomize, не применяя их к кластеру.

1.  **Создайте новую ветку** от `development`.
2.  **Внесите небольшое изменение** в любой файл в директории `deploy/k8s/` (например, добавьте комментарий).
3.  **Сделайте коммит и пуш** в вашу новую ветку.
4.  **Создайте Merge Request** в ветку `development`.
5.  После мержа вашего запроса автоматически запустится пайплайн для ветки `development`.
6.  **Откройте GitLab -> CI/CD -> Pipelines** и найдите ваш пайплайн.
7.  Найдите задачу `dry-run-dev` на этапе `validate`. **Изучите ее лог**:
    *   Убедитесь, что команда `kubectl apply --dry-run=client` завершилась успешно.
    *   Просмотрите отрендеренный YAML, который вывела команда `kubectl kustomize`. Это именно то, что будет отправлено в кластер на следующем этапе.

    ✅ **Результат**: Если задача `dry-run-dev` прошла успешно, значит, ваши Kustomize-конфигурации и манифесты синтаксически корректны.

### Б. Тестирование этапа развертывания (`deploy`)

Этот этап выполняется только после успешного завершения `validate`.

1.  После успешного выполнения задачи `dry-run-dev`, автоматически начнется выполнение этапа `deploy`.
2.  Найдите в пайплайне задачу `deploy-dev`.
3.  **Изучите ее лог**: убедитесь, что команда `kubectl apply -k` выполнилась без ошибок.

    ✅ **Результат**: Если задача `deploy-dev` прошла успешно, ваше приложение развернуто в dev-окружении Kubernetes.

## 2. Тестирование развертывания в локальном кластере

Эти шаги позволяют полностью развернуть и проверить приложение на вашем локальном Kubernetes-кластере (Minikube, Docker Desktop и т.д.).

### Шаг 1: Подготовка `.env` файла
- В корне проекта находится файл `.env`.
- Убедитесь, что в нем указаны корректные данные для подключения к вашей **локальной** базе данных (`DB_USER`, `DB_PASSWORD`, `DB_NAME`).

### Шаг 2: Предварительный просмотр манифестов
- Перед применением конфигурации вы можете посмотреть, что именно будет создано в кластере.
- Выполните из корня проекта:
  ```shell
  kubectl kustomize deploy/k8s/overlays/local
  ```
- Эта команда выведет в консоль все готовые манифесты. Проверьте, что в сгенерированном секрете `s5-db-credentials` есть поля `DB_USER`, `DB_PASSWORD` и `DB_NAME`.

### Шаг 3: Развертывание
1.  **Разверните кластер базы данных**:
    ```shell
    kubectl apply -f deploy/k8s/k8s-db-cluster.yaml
    ```
2.  **Разверните приложение**:
    ```shell
    kubectl apply -k deploy/k8s/overlays/local/
    ```

### Шаг 4: Проверка
1.  **Проверьте статус подов**:
    ```shell
    kubectl get pods
    ```
    Убедитесь, что под вашего приложения (`s5-server-deployment-...`) имеет статус `Running`.

2.  **Просмотрите логи приложения**:
    ```shell
    # Замените <имя-вашего-пода> на реальное имя из команды выше
    kubectl logs <имя-вашего-пода>
    ```
    Ищите ошибки подключения к базе данных или другие проблемы при старте.

3.  **Пробросьте порт для доступа к приложению**:
    ```shell
    kubectl port-forward service/s5-server-service 8080:80
    ```
4.  Откройте в браузере `http://localhost:8080`, чтобы увидеть работающее приложение.

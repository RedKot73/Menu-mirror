## 📤 Передача клиенту

### Вариант 1: ZIP архив (рекомендуется)

```powershell
# После успешной сборки
Compress-Archive -Path "Release\win-x64\*" -DestinationPath "S5Server.zip"
```

### Вариант 2: Прямая копия директории

Скопируйте всю папку `Release\win-x64\` на USB или сетевой диск

---

## 🖥️ Запуск у клиента

### Windows

1. Распаковать архив в любую папку (например: `C:\S5Server\`)
2. Запустить `start.bat` (двойной клик)
3. Открыть браузер: http://localhost:5000

### Если порт 5000 занят

1. Открыть `appsettings.Production.json`
2. Изменить `"Url": "http://0.0.0.0:5000"` на другой порт
3. Сохранить и запустить `start.bat`

## ⚙️ Конфигурация продакшена

### Изменить порт

Файл: `appsettings.Production.json`

```json
{
  "Kestrel": {
    "Endpoints": {
      "Http": {
        "Url": "http://0.0.0.0:8080"  ← Измените порт
      }
    }
  }
}
```

### Изменить путь к базе данных:

```json
{
  "ConnectionStrings": {
    "DefaultConnection": "Data Source=D:/MyData/S5_DB.sqlite"
  }
}
```

## 🔄 Обновление у клиента

### Простое обновление (без изменений в БД):

1. Остановить приложение (Ctrl+C в консоли или закрыть окно)
2. **BACKUP:** Скопировать `Data\S5_DB.sqlite` в безопасное место
3. Заменить все файлы КРОМЕ:
   - `Data\` (база данных)
   - `appsettings.Production.json` (настройки клиента)
4. Запустить `start.bat`

### Полное обновление:

1. Остановить приложение
2. Сделать полный backup директории
3. Удалить старые файлы
4. Распаковать новый релиз
5. Скопировать `Data\S5_DB.sqlite` из backup
6. Скопировать настройки из старого `appsettings.Production.json`
7. Запустить `start.bat`

## ❗ Troubleshooting

### Проблема: "Порт 5000 уже используется"

**Решение:** Измените порт в `appsettings.Production.json`

### Проблема: "Не удается запустить приложение"

**Решение:**

- Запустите `start.bat` от имени администратора
- Проверьте брандмауэр Windows
- Проверьте антивирус (добавьте в исключения)

### Проблема: "База данных недоступна"

**Решение:**
- Проверьте наличие файла `Data\S5_DB.sqlite`

- Проверьте права доступа к файлу

### Проблема: "Белый экран в браузере"

**Решение:**
- Проверьте наличие папки `wwwroot`
- Очистите кеш браузера (Ctrl+F5)
- Проверьте консоль браузера (F12)

## 🔒 Безопасность

### Рекомендации:

- ✓ Регулярно создавайте backup базы данных

```powershell

Copy-Item "Data\S5_DB.sqlite" "Data\S5_DB_backup_$(Get-Date -Format 'yyyyMMdd_HHmmss').sqlite"
```

# Project Map

Этот файл предоставляет контекст о проекте для ИИ-агентов. Он описывает архитектуру, технологический стек, структуру папок, правила безопасности и инфраструктуру (включая CI/CD и Kubernetes).


## Архитектура
Проект представляет собой монолитное веб-приложение, где backend обслуживает как REST API (если есть гибридное API), так и GraphQL эндпоинты, а также сервирует собранный Frontend.
- **Backend (Паттерны)**: 
  - Используется "Тонкая" архитектура, без классического CQRS (MediatR) или избыточного Repository Pattern. 
  - GraphQL резолверы на `HotChocolate` получают `[Service] MainDbContext` напрямую и используют возможности `[UseProjection]`, `[UseFiltering]`, `[UseSorting]` для гибкой трансляции GraphQL-запросов прямо в IQueryable и SQL (Entity Framework Core).
  - Бизнес-логика или сложные запросы вынесены в директорию `Services/`, где реализованы в виде *Extention Methods* поверх `DbSet<T>` (например, формирование кастомных Union запросов через `AsNoTracking()`).
- **Frontend**: Встроенное SPA-приложение, исходный код которого находится в папке `Front/` (Angular). При сборке (через Target в `.csproj`) Angular-дистрибутив полностью копируется в публикуемую директорию backend-a (`wwwroot`).
- **Database**: Реляционная СУБД, доступ к которой осуществляется через EF Core с использованием миграций (`Migrations/`).

## Frontend Architecture
**Core Tech**: Angular 21.0.5 (SPA). Проект **НЕ использует** Razor (`.cshtml`) или Blazor (`.razor`). Все современные фичи Angular (сигналы, управляющие конструкции `@if`/`@for`) приветствуются.
**UI Framework**: Только **Angular Material**. Использование Bootstrap (например, классы вроде `btn btn-primary`) или Tailwind запрещено без отдельного указания! Вместо этого используй `<button mat-raised-button color="primary">`.

**Project Structure**:
- Корень фронтенда: `/S5Server/Front/src/`
- **Главный Layout**: `app.html` (содержит `<app-navigator />` и `<router-outlet />`).
- **Навигация**: `navigator.component.ts` (меню реализовано через `<mat-toolbar>`).

**Styling Rules**:
- **Глобальные стили**: `styles.css` и `custom-theme.scss`.
- **Шрифты/Иконки**: Roboto и Material Icons (подключены через CDN в `index.html`).
- **Локальные стили**: В Angular стили часто инкапсулированы внутри компонентов. При изменении верстки обязательно проверяй наличие стилей напрямую внутри декоратора `@Component` (в свойстве `styles`) в `.ts` файлах (например, как в `navigator.component.ts`).

**Workflow**:
- Для визуальных правок используй `dotnet watch` (т.к. бэкенд хостит статику) или команды Angular CLI (например, `ng start`), если они доступны.
- Всегда используй компоненты Angular Material для соблюдения единообразия дизайна.
## Стек
### Backend
- **Фреймворк**: .NET 10 (ASP.NET Core)
- **API**: GraphQL (HotChocolate 15.1.x)
- **ORM**: Entity Framework Core 10
- **Аутентификация**: ASP.NET Core Identity + JWT (Microsoft.AspNetCore.Authentication.JwtBearer)
- **Логирование**: Serilog (с интеграцией Application Insights, Console, File)
- **Документооборот**: OpenXML для работы с документами (БР/БД)

### Frontend
- **Фреймворк**: Angular 
- **Сборка**: Angular-билд публикуется напрямую в директорию `wwwroot` для сервировки через ASP.NET Core.

### База Данных
- **СУБД**: PostgreSQL 
- **Адаптер**: Npgsql.EntityFrameworkCore.PostgreSQL

## Структура
Ключевые директории в `S5Server/`:
- `Controllers/` — REST контроллеры.
- `GraphQL/` — Резолверы, мутации, типы для HotChocolate GraphQL.
- `Data/` & `Models/` — EF Core контексты и доменные модели (Entities).
- `Migrations/` — Миграции EF Core для PostgreSQL.
- `Services/` — Сервисы бизнес-логики приложения.
- `Front/` — Исходный код Frontend (Angular).
- `Utils/` — Утилиты и вспомогательные классы.

Ключевые файлы:
- `S5Server/S5Server.csproj` — Основной файл проекта, содержит конфигурации `net10.0` и список зависимостей NuGet.
- `S5Server/appsettings.json` — Основной конфигурационный файл приложения.
- `S5Server/Program.cs` — Точка входа в приложение и настройка DI-контейнера / Middleware пайплайна.
- `.github/workflows/build-release.yml` — Файл пайплайна для сборки self-contained кроссплатформенных релизов.

## Правила безопасности
- **Кросс-доменные запросы**: Настроен строгий CORS (в `appsettings.json` задан список `AllowedOrigins`).
- **Авторизация**: Токены JWT используются для аутентификации. Подключены библиотеки `Microsoft.AspNetCore.Authentication.JwtBearer` и `HotChocolate.AspNetCore.Authorization`.
- **Секреты**: В разработке используется `UserSecretsId` (`ef5aa0a8-6487-4a74-9cf5-ef0d2e7090bd`). В продакшене (Azure/K8s) секреты и ключи подтягиваются через `Azure.Extensions.AspNetCore.Configuration.Secrets` или переменные окружения K8s.

## Инфраструктура
*Согласно #AI/__cicd.txt:*

- **Kubernetes & СУБД**: Развертывание PostgreSQL управляется через оператор **CloudNativePG (CNPG)**. Это обеспечивает высокую доступность, автоматический failover и отказоустойчивость базы данных в кластере Kubernetes.
- **Сборка БД**: Для специфической инициализации и создания кастомного образа базы данных используется файл `Dockerfile.kuber`.
- **CI/CD Flow**: В репозитории (через GitHub Actions) настроена сборка self-contained релизов для Windows, Linux и macOS с использованием кроссплатформенных архивов. В рамках Kubernetes используются pipeline'ы (описанные в __cicd.txt) для автоматического деплоя этих артефактов / образов в кластер.
- **Облако**: Интеграция с Azure (AppServices / Application Insights / подключение осуществляется через `Azure.Identity`).
- **.env и Конфигурация в K8s**: Конфигурация приложения берется из `appsettings.json` и гибко переопределяется через переменные окружения в K8s (соответствующие `.env` файлу). Например, параметры БД в блоке `PrimaryConnection`:
  В Kubernetes задаются переменные среды (Environment Variables) в формате:
  - `PrimaryConnection__DB_Host`
  - `PrimaryConnection__DB_Name`
  - `PrimaryConnection__Port`
  - `PrimaryConnection__CommandTimeout`
  - `PrimaryConnection__SearchPath` (По умолчанию: `core,dict,docs,identity,history,public`)

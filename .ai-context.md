# Project Ground Truth

This document serves as the absolute source of truth for AI agents interacting with this project. It outlines the system architecture, technology stack, directory structure, navigation mapping, security rules, and infrastructure.

## Architecture Guidelines
The system is built as a highly coupled monolith, where the .NET Backend simultaneously serves a REST API, acts as a GraphQL Server, and hosts the compiled Angular Frontend.

- **"Thin" Backend Architecture**: Avoid complex architectural patterns like `CQRS (MediatR)` or `Repository Pattern`. Focus heavily on using `Entity Framework Core` directly in conjunction with `HotChocolate` via attributes (`[UseProjection]`, `[UseFiltering]`, `[UseSorting]`) to stream data effortlessly.
- **Business Logic Layer**: Complex queries or operations reside in `Services/` and should be implemented as **Extension Methods** layered on top of `DbSet<T>`.
- **Frontend SPA**: Powered by Angular 21.0.5. Source code lives in `/S5Server/Front/src/`. Upon building (via MSBuild Targets in the `.csproj`), the Angular app compiles straight into the backend's `wwwroot` directory to be statically served. **DO NOT** use Razor (`.cshtml`) or Blazor (`.razor`).
- **UI Framework Rules**: **Angular Material ONLY**. The use of Tailwind CSS, Bootstrap, or any custom utility-first framework is strictly forbidden unless explicitly mandated by the user. Rely on native Angular Material properties and components.
- **Database**: Relational data relies entirely on **PostgreSQL** configured via `Npgsql.EntityFrameworkCore.PostgreSQL`, with Code-First schema management via EF Core Migrations (`Migrations/`).

## Naming Conventions
To maintain uniformity across the monorepo, strictly adhere to the following rules:
- **C# (Backend)**: Use `PascalCase` for namespaces, classes, records, properties, methods, and interfaces (with `I` prefix). Use `camelCase` for variables and method parameters.
- **TypeScript/Angular (Frontend)**: Use `camelCase` for variable names, component properties, and functions. Use `PascalCase` for Class and Interface names. Strictly adhere to Angular file naming schemes (`kebab-case.component.ts`, `kebab-case.service.ts`).
- **GraphQL API**: Use `camelCase` for all Queries, Mutations, and property accessors, standardizing JSON responses.

## Security Constraints
- **CORS Policy**: Configured using `AllowedOrigins` from `appsettings.json`. Wildcard origin requests are blocked on production, allowing exceptions only for localized development servers (`localhost:4200`).
- **Authorization Schema**: Access uses JWT Bearer Tokens validated by `HotChocolate.AspNetCore.Authorization` middleware. APIs enforce HTTP `401/403` status outputs in lieu of redirecting automatically.
- **Secrets Management**: Local environment depends entirely on `UserSecretsId`. Staging/Production clusters mount secrets dynamically utilizing Kubernetes mapped Environmental Variables (`.env`) or Azure Identity modules.

## Infrastructure & DevOps
- **Kubernetes (K8s) & RDBMS**: The PostgreSQL database cluster runs inside Kubernetes, governed seamlessly by the **CloudNativePG (CNPG)** operator. This grants highly available environments, replication, and proactive error failover.
- **Containerization**: Database initialization leverages a localized Docker image (`Dockerfile.kuber`).
- **Continuous Integration (CI/CD)**: GitHub Actions automates artifact builds via pipeline deployments (`.github/workflows/build-release.yml`), compiling self-contained cross-platform binaries. Custom pipeline workflows translate the artifacts automatically into K8s cluster updates.
- **Telemetry & Logging**: The server relies solely on **Serilog** for centralized intelligence. It integrates natively with Azure Application Insights (`Microsoft.ApplicationInsights.AspNetCore`) for production analysis, whilst dropping localized diagnostic logs into `Logs/s5server-.txt`.
- **Environment Configuration**: K8s seamlessly bridges runtime variables to application settings. For example, standard `.env` configuration cascades gracefully into hierarchical `.NET` configurations (e.g. `PrimaryConnection__DB_Host` dynamically overwrites the `PrimaryConnection` -> `DB_Host` JSON element).
